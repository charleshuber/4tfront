Projet SMB215 (2017) Charles Huber

Réalisation d’une application répartie sous Docker


Pour le moment, seulement une gestion basique des utilisateurs a été implémentée...


L'authentification, la gestion des droits et le protocole https seront mis en place ultérieurement !


Le fonctionnement de l'architecture docker est validée par l'implémentation du CRUD (Create Read Update Delete) sur la ressource "utilisateur". Ceci sur l'ensemble de la chaine applicative respectant le "design-pattern" MVC:
  • Modèle : Conteneur Mysql
  • Vue : Conteneur "4tfront" hébergeant le serveur Apache et le code Angular2 du client
  • Controleur : Conteneur "4trest" hébergeant le tomcat qui expose les services REST

Ce projet a vocation a mettre en place les briques de base de mon projet de mémoire. Il s'agit d'une application de gestion des congés et du temps de travail nommée "tatoutontan". (tttt => 4t)
